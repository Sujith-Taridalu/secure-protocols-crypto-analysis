# Executive Summary

## Project Purpose

This project presents a curated and sanitized portfolio of secure-protocol and cryptography analysis work. It converts coursework and lab notes into public-safe security documentation that demonstrates conceptual depth without exposing raw assignments, personal identifiers, packet captures, or sensitive network details.

## Scope

The project covers five major areas:

1. Cryptography foundations
2. PKI and digital certificates
3. TLS/SSL handshake behavior
4. IPSec and secure packet handling
5. Network and wireless security reasoning

## Key Outcomes

| Area | Outcome |
|---|---|
| Cryptography | Explained encoding, classical ciphers, RSA, DES/3DES, CBC vs ECB, MACs, signatures, and PGP ordering. |
| PKI | Analyzed certificate chains, CA trust, revocation locations, signature algorithms, and public-key properties. |
| TLS | Interpreted handshake records, content types, cipher-suite negotiation, random values, and encrypted application data. |
| IPSec | Explained anti-replay protection, one-way Security Associations, application transparency, and Encrypt-then-MAC. |
| Network Security | Interpreted traceroute behavior, DNS reflection risk, service discovery, and switched-LAN defensive controls. |
| Wireless Security | Explained why WEP is obsolete and why WPA2/WPA3-style protections are needed. |

## Portfolio Value

This repository is strongest for demonstrating security reasoning rather than software development. It shows the ability to:

- explain complex security mechanisms clearly;
- connect cryptographic primitives to real protocol behavior;
- identify weaknesses and defensive mitigations;
- document analysis in a professional, public-safe format;
- avoid exposing sensitive or academic-only artifacts.
